# Prompt 30 Status

## Scope Delivered

Prompt 30 delivered the **Synthetic Demo/UAT Data Foundation** requested by supervisor feedback.

This phase added explicit seed, reset, and verify support for meaningful synthetic demo/UAT data across the existing Phase 1 ERP modules. It did not add new frontend charts, dashboard widgets, business modules, CRUD screens, reports, workflows, migrations, production deployment changes, or automatic startup seeding.

## Commands Added

```powershell
corepack pnpm seed:demo
corepack pnpm seed:demo:reset
corepack pnpm seed:demo:verify
```

Useful safety commands:

```powershell
corepack pnpm seed:demo -- --dry-run
corepack pnpm seed:demo:reset -- --dry-run
```

## Synthetic Target

The seed creates or reuses only the clearly named synthetic company:

```text
Real Capita Demo / UAT
real-capita-demo-uat
```

It creates synthetic demo/UAT users under:

```text
demo.realcapita.test
```

## Data Coverage

Seed coverage includes:

- Org & Security: synthetic company, demo users, role assignments, locations, departments
- Accounting: account hierarchy, posting accounts, posted and draft vouchers, all Phase 1 voucher types
- Financial reports: posted activity across asset, liability, equity, revenue, and expense accounts
- Project/property: projects, cost centers, phases, blocks, zones, unit types, units with status variation
- CRM/property desk: customers, leads, bookings, sale contracts, installment schedules, collections linked to posted vouchers
- HR: employees, attendance devices, device mappings, attendance logs, leave types, leave requests with lifecycle variation
- Payroll: salary structures, draft/finalized/posted payroll runs, payroll run lines, payroll posting through the existing database function
- Audit/documents: safe synthetic attachment metadata, attachment links, and audit events

## Safety Preserved

- No automatic seeding during startup, Docker startup, migrations, or normal bootstrap.
- Reset is scoped to the exact synthetic company slug and expected synthetic company name.
- Reset refuses if unmarked records exist in the synthetic company.
- Reset uses `DEMO-`, `UAT-`, and `SYNTH-DEMO-UAT` markers to distinguish seed-created records.
- Auth audit events generated by synthetic `demo.realcapita.test` users are accepted as synthetic demo/UAT operational artifacts for reset safety.
- Production mode requires explicit `--confirm-production-demo-data`.
- Existing real/user-created company data is not targeted.
